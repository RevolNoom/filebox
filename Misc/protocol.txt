Ghi chú:
	METADATA:
	+) Định dạng:
	Tên_file:Kích_Cỡ_CHÍNH_XÁC_THEO_BYTE
	+) Ví dụ:
	presentation.pdf:3500 (file tên là presentation.pdf, kích thước 3500 byte (3.5KB))

	\n là kết thúc dòng. Được sử dụng để kết thúc các packet


I. Client upload lên server
Client gửi: "up /thư_mục_đến/trên_server/ METADATA\n"
Server gửi: "ok_up\n" nếu không có lỗi. "no_up ERROR_CODE\n" nếu có lỗi xảy ra
Client bắt đầu truyền tệp.
Server dừng tiếp nhận thêm gói tin khi đã nhận đủ kích thước theo metadata



II. Client download từ server
Client gửi: "down /thư_mục/trên_server/tên_file\n"
Server gửi: "ok_down METADATA\n" nếu không có lỗi. "no_down ERROR_CODE\n" nếu có lỗi xảy ra
Client gửi: "ok_down\n" nếu không có lỗi. "no_down\n" nếu có lỗi xảy ra (hết bộ nhớ trên máy chẳng hạn).
Server bắt đầu truyền tệp.
Client dừng tiếp nhận thêm gói tin khi đã nhận đủ kích thước theo metadata



III. Client nhập lệnh ls:
Màn hình hiện lên danh sách tệp tin cũ
Song song với đó, client gửi: "ls thời_gian_hiện_tại\n"
Server gửi "ok_ls\n" nếu hệ thống tập tin chưa thay đổi từ lúc client gửi
Server gửi lại cây hệ thống tập tin nếu cây hệ thống tập tin đã thay đổi trước thời_gian_hiện_tại.
Giả sử Cây hệ thống tập tin trên server đang như này:
/
|_ dir1
|_ dir2
   |_ file2
|_ file1

Thì tin nhắn server gửi về sẽ như sau:
update_ls file1 dir1/ dir2/file2\n 
Có thể gửi theo thứ tự khác nhau. Ví dụ như "update_ls  dir1/ dir2/file2 file1\n". Miễn là in đầy đủ
đường dẫn tuyệt đối của file.

Lưu ý: Nếu cây hệ thống tập tin hiu quạnh chả có file, thư mục nào, thì tin nhắn gửi về chỉ vỏn vẹn:

update_ls\n



IV. Client đăng ký tài khoản.
Sau khi UI đã kiểm tra hai mật khẩu giống nhau, không cần mã hóa tập tin, packet gửi cho server có dạng:

reg tên_người_dùng\nmật khẩu\n

Ví dụ người dùng đăng ký với tên An, mật khẩu abc, thì gói tin client gửi cho server sẽ là:

reg An:abc\n

Server trả về "ok_reg\n" nếu không có lỗi, "no_reg ERROR_CODE\n" nếu có lỗi xảy ra. Ví dụ như trùng tên với người khác.



V. Client đăng nhập
Sau khi người dùng nhập tên và mật khẩu vào UI xong, gửi tin cho server dưới dạng plaintext:

log tên_người_dùng:mật khẩu\n

Ví dụ người dùng đăng nhập với tên An, mật khẩu abc, thì gói tin client gửi cho server sẽ là:

log An:abc\n

Server trả về "ok_log\n" nếu không có lỗi, "no_log ERROR_CODE\n" nếu có lỗi xảy ra.



VI. Client xóa file
Client gửi: "del /đường/dẫn/file/trên/server\n"
Server gửi lại "ok_del\n" nếu thành công. "no_del ERROR_CODE\n" nếu có lỗi (file không tồn tại chẳng hạn)
